You are given two arrays of length n:

- cost, where cost[i] is the cost of placing a filter on server i
- nextServer, where nextServer[i] is the server that a packet moves to from server i

Each server sends a packet to exactly one server, so starting from any server and repeatedly following nextServer will eventually enter a cycle.

A packet is considered caught if it reaches a server that has a filter installed.

Choose a set of servers on which to install filters such that a packet starting from any server is guaranteed to be caught eventually.

Return the minimum total cost required.

Notes:
- Servers are numbered from 1 to n.
- Since every server has exactly one outgoing edge, the graph is made of directed chains leading into directed cycles.
- To guarantee that all packets are caught, every directed cycle must contain at least one filtered server.

Example 1:
Input:
cost = [5, 2, 7, 10]
nextServer = [2, 3, 1, 3]

Output:
2

Explanation:
Servers 1, 2, and 3 form a cycle: 1 -> 2 -> 3 -> 1.
Server 4 leads into that cycle.
Placing a filter on server 2 is enough to catch packets starting from any server.
The minimum cost in that cycle is 2, so the answer is 2.

Example 2:
Input:
cost = [8, 4, 6, 3, 5, 2]
nextServer = [2, 1, 4, 5, 3, 5]

Output:
7

Explanation:
There are two cycles:
- 1 -> 2 -> 1, with minimum filter cost 4
- 3 -> 4 -> 5 -> 3, with minimum filter cost 3
Server 6 leads into the second cycle.
So the minimum total cost is 4 + 3 = 7.

Constraints:
- 1 <= n <= 2 * 10^5
- 1 <= cost[i] <= 10^9
- 1 <= nextServer[i] <= n

You are given an integer N representing N microservices labeled from 0 to N - 1.

For each service i, you are given a list deps[i] containing the services that must already be started before service i can start.

The system tries to start services in repeated cycles:

- In each cycle, services are checked in order from 0 to N - 1.
- If all dependencies of a service have already been started, that service starts immediately.
- Otherwise, it is skipped for that cycle.
- The process repeats until all services are started.

A service started earlier in the same cycle can satisfy dependencies for services checked later in that same cycle.

Return the minimum number of cycles required to start all services.
If it is impossible to start all services because of cyclic dependencies, return -1.

Example 1:
Input:
N = 2
deps = [[1], []]

Output:
2

Explanation:
Cycle 1:
- Service 0 cannot start because service 1 is not started yet.
- Service 1 starts.

Cycle 2:
- Service 0 now starts.

Example 2:
Input:
N = 2
deps = [[], [0]]

Output:
1

Explanation:
Cycle 1:
- Service 0 starts.
- Service 1 starts later in the same cycle because service 0 is already started.

Example 3:
Input:
N = 3
deps = [[1], [0], []]

Output:
-1

Explanation:
Services 0 and 1 depend on each other, so it is impossible to start all services.

Constraints:
- 1 <= N <= 2 * 10^5
- 0 <= deps[i].length <= N - 1
- The total number of dependencies across all services is at most 2 * 10^5
- All dependency indices are in the range [0, N - 1]
- A service does not depend on itself

Function signature:
int minCycles(int N, vector<vector<int>>& deps);
